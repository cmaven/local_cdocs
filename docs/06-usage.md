# 06. 사용자 가이드

## 설치

### NSIS 설치본 (권장)

1. `local_cdocs-x.x.x-Setup.exe` 파일을 다운로드합니다.
2. 파일을 더블클릭하여 설치 마법사를 실행합니다.
3. 설치 경로를 선택하고 "설치" 버튼을 클릭합니다.
4. 설치 완료 후 바탕화면 또는 시작 메뉴의 **local_cdocs** 아이콘으로 실행합니다.

> Node.js를 별도로 설치할 필요가 없습니다. 모든 런타임이 앱에 내장되어 있습니다.

### Portable (설치 없이 사용)

1. `local_cdocs-x.x.x-Portable.exe` 파일을 다운로드합니다.
2. 원하는 위치에 저장 후 더블클릭하여 바로 실행합니다.
3. USB 드라이브에 저장해 어느 PC에서나 사용할 수 있습니다.

---

## 첫 실행 — 관리형 라이브러리

앱은 **고정 라이브러리 폴더**를 항상 엽니다:

```
C:\Users\<사용자>\Documents\local-cdocs
```

처음 실행하면 이 폴더가 **자동으로 생성**되고, `시작하기` 컬렉션(사용법 안내)이 시드됩니다.
홈 화면은 **라이브러리 대시보드**로, 라이브러리 안의 컬렉션들이 카드로 표시됩니다.

> 임의 폴더를 여는 방식(`폴더 열기`)은 더 이상 사용하지 않습니다.
> 모든 문서는 위 라이브러리 폴더 아래에 **컬렉션 폴더**로 둡니다(아래 "문서 폴더 구조" 참고).

---

## 컬렉션 관리 (대시보드)

홈 화면(대시보드)에서 컬렉션을 직접 관리합니다. **컬렉션 = 라이브러리 안의 폴더 하나 = 카테고리**입니다.

| 동작 | 위치 | 설명 |
|------|------|------|
| **+ 새 컬렉션** | 상단 액션바 | 이름 입력 모달 → 라이브러리에 새 폴더(카테고리) 생성 |
| **폴더 가져오기** | 상단 액션바 | 외부의 `.md` 폴더를 선택 → 라이브러리로 **복사**해 컬렉션으로 추가 |
| **라이브러리 폴더 열기** | 상단 액션바 | `Documents\local-cdocs`를 Windows 탐색기로 엶 |
| **이름변경 / 삭제 / 탐색기** | 각 카드의 `⋯` 메뉴 | 이름변경(모달), 삭제(휴지통으로 이동), 해당 폴더를 탐색기로 열기 |

- 같은 동작이 상단 메뉴 **파일**(새 컬렉션 `Ctrl+N` / 폴더 가져오기 / 라이브러리 폴더 열기)에도 있습니다.
- 컬렉션을 만들거나 가져오거나 바꾸면 약 2초 후 화면이 **자동으로 갱신**됩니다.
- 탐색기에서 직접 폴더/`.md`를 추가·삭제해도 동일하게 자동 반영됩니다.

## 문서 폴더 구조

**최상위의 "아무 폴더명"이나 카테고리가 됩니다** (연도 같은 특별한 이름 불필요).
폴더 이름이 곧 상단 드롭다운의 카테고리이자 사이드바 그룹이 됩니다.

### 기본 구조 (폴더 = 카테고리)

```
내 문서 폴더/
├── index.md                ← (선택) 홈 화면. 없으면 앱이 자동 생성
├── 관리자가이드/            ← 카테고리 "관리자가이드"
│   ├── index.md            ← "개요"로 맨 앞 + 홈 카드 제목/설명(frontmatter)
│   ├── 설치.md
│   └── 운영.md
├── api/                    ← 카테고리 "Api"
│   ├── index.md
│   └── endpoints.md
└── guide/                  ← 카테고리 "Guide"
    └── index.md
```

- 폴더명에 `-`나 `_`가 있으면 표시할 때 공백+첫글자 대문자로 다듬어집니다 (`local_docs` → `Local Docs`).
- `index.md`의 frontmatter `title`/`description`은 홈 카드와 사이드바 라벨에 사용됩니다.

### 하위 폴더 = 접이식 하위 그룹

```
내 문서 폴더/
└── notes/                  ← 카테고리 "Notes"
    ├── intro.md
    └── 2024/               ← 접이식 하위 그룹 "2024"
        └── jan.md
```

### 평면 구조도 지원 (폴더 없이 .md만)

```
내 문서 폴더/
├── index.md
├── chapter1.md             ← 루트에 그냥 .md를 둬도 사이드바에 표시
└── chapter2.md
```

> 참고: 기존 cdocs의 `연도(2026)/프로젝트/파일` 구조도 그대로 동작합니다 —
> 이 경우 연도가 카테고리, 프로젝트가 하위 그룹으로 보입니다.

## .md 파일 추가

폴더에 `.md` 파일을 추가하면 **자동으로** 사이드바와 홈 화면에 반영됩니다.

```bash
# 예: Windows 탐색기에서 파일 생성 또는 편집기에서 저장
새 파일 → 내 문서 폴더/api/new-page.md
```

약 2초 후 사이드바가 갱신되고 화면이 자동으로 새로고침됩니다.

> **자동 갱신이 안 될 때:** 상단 메뉴 → **보기** → **새로고침** (`Ctrl+R`)으로 수동 새로고침할 수 있습니다.

## 지원 기능

### Mermaid 다이어그램

Markdown 코드블록에 `mermaid` 언어를 지정하면 다이어그램으로 렌더링됩니다.

````md
```mermaid
graph LR
  A[시작] --> B[처리] --> C[완료]
```
````

flowchart, sequenceDiagram, gantt, classDiagram, gitgraph 등 Mermaid가 지원하는 모든 유형을 사용할 수 있습니다.

---

### 로컬 검색

우측 상단 검색 아이콘(또는 `Ctrl+K`)을 클릭하면 현재 폴더의 모든 `.md` 파일을 대상으로 전문 검색(full-text search)합니다. 인터넷 연결 없이 로컬에서만 동작합니다.

---

### 다크 모드

우측 상단의 다크 모드 토글 버튼으로 라이트/다크 테마를 전환합니다. 선택한 테마는 앱 재시작 후에도 유지됩니다.

---

### 커스텀 컴포넌트

`.md` 파일 안에서 아래 컴포넌트를 HTML 태그처럼 사용할 수 있습니다.

#### Tabs / Tab — 탭 패널

```md
<Tabs>
  <Tab label="macOS">brew install ...</Tab>
  <Tab label="Windows">winget install ...</Tab>
</Tabs>
```

#### Steps / Step — 단계별 가이드

```md
<Steps>
  <Step title="저장소 클론">git clone ...</Step>
  <Step title="의존성 설치">npm install</Step>
  <Step title="실행">npm run dev</Step>
</Steps>
```

#### Callout / Hint — 강조 박스

```md
<Callout type="info">정보 메시지</Callout>
<Callout type="warning">경고 메시지</Callout>
<Callout type="danger">위험 메시지</Callout>
<Hint type="tip">팁 메시지</Hint>
```

#### Badge — 인라인 배지

```md
<Badge type="tip" text="NEW" />
<Badge type="warning" text="DEPRECATED" />
```

#### Details / Accordion — 접기/펼치기

```md
<Details summary="자세히 보기">
숨겨진 내용이 여기 들어갑니다.
</Details>
```

#### Columns / Column — 다단 레이아웃

```md
<Columns>
  <Column>왼쪽 내용</Column>
  <Column>오른쪽 내용</Column>
</Columns>
```

---

### frontmatter 활용

`.md` 파일 상단에 frontmatter를 추가하면 사이드바 타이틀과 홈 카드 설명을 제어할 수 있습니다.

```md
---
title: 나의 프로젝트 개요
description: 이 프로젝트에 대한 간략한 설명입니다.
---

# 본문 시작
```

- `title`: 사이드바 항목 이름 및 브라우저 탭 제목
- `description`: 홈 화면 프로젝트 카드의 설명 텍스트

## 알려진 제한사항

- **이미지 경로**: 절대 경로(`/images/foo.png`)는 VitePress의 `public/` 디렉토리를 기준으로 합니다. 사용자 폴더에 `public/` 하위 디렉토리를 만들어 이미지를 저장하세요.
- **Windows 한글 폴더명**: 대부분의 경우 정상 동작하지만, 특수문자가 포함된 경로에서 문제가 발생할 수 있습니다.
- **서버 기동 시간**: 폴더를 처음 열거나 파일 변경 후 사이드바 갱신까지 약 2~5초가 소요됩니다.
- **외부 이미지**: 인터넷 연결이 없는 환경에서는 외부 URL 이미지가 표시되지 않습니다. 이미지는 문서 폴더 내에 저장하는 것을 권장합니다.
