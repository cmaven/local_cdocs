# local-cdocs 사용법

**local-cdocs**는 cdocs 문서 포털을 그대로 데스크톱으로 옮긴 로컬 Markdown 뷰어입니다.
폴더만 선택하면 그 안의 `.md` 파일이 웹 cdocs와 동일한 화면으로 렌더링됩니다.

## 시작하기

1. 상단 메뉴 **파일 > 폴더 열기** (`Ctrl+O`)를 선택합니다.
2. 문서가 들어있는 폴더를 고릅니다.
3. 사이드바·홈 화면이 폴더 구조에 맞춰 자동 생성됩니다.

다음 실행부터는 마지막에 열었던 폴더가 자동으로 로드됩니다.

## 폴더 구조 규칙

cdocs와 동일한 규칙을 따릅니다.

```
선택한_폴더/
├── index.md            # 랜딩 페이지 (없으면 자동 생성: <HomePage />)
├── guide/              # 가이드 문서
│   └── index.md
├── 2026/               # 연도 디렉토리 (4자리 숫자)
│   └── project-alpha/  # 프로젝트
│       ├── index.md    # "개요"로 사이드바 맨 앞 배치
│       └── design.md
└── 2025/
    └── project-beta/
        └── index.md
```

- **연도 디렉토리**(`2020`~`2026` 등 4자리 숫자) 안의 프로젝트가 자동으로 사이드바·홈 카드로 등록됩니다.
- 각 프로젝트의 `index.md`는 "개요"로 맨 앞에 표시됩니다.
- `index.md`의 frontmatter `title`/`description`이 카드 제목/설명에 반영됩니다.
- `project-alpha`, `project-alpha-v2`처럼 `-v2` 접미사가 붙으면 버전 그룹으로 묶입니다.

## 실시간 자동 갱신

폴더 안에서 `.md` 파일이나 하위 디렉토리를 **추가/삭제**하면,
약 2초 뒤 사이드바와 홈 화면이 자동으로 갱신됩니다. 앱을 다시 시작할 필요가 없습니다.

## 지원 기능

- 자동 사이드바 / 홈 카드 생성
- Mermaid 다이어그램 (` ```mermaid ` 코드블록)
- 로컬 전문 검색
- 다크 모드 (`Ctrl+D` 또는 우상단 토글)
- 커스텀 컴포넌트: `<Callout>`, `<Tabs>`, `<Steps>`, `<Columns>`, `<Badge>`, `<Asciinema>` 등

## 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+O` | 폴더 열기 |
| `Ctrl+R` | 새로고침 |
| `Ctrl+D` | 다크 모드 토글 |
| `Ctrl+Q` | 종료 |
